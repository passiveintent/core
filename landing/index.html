Where it fits

A small in-browser layer for flows where hesitation is expensive.

Use it in checkout, pricing, billing, onboarding, and support-sensitive journeys where acting in-session matters more than post-hoc analytics.

The shipping product is PassiveIntent Core. JavaScript and React packages are available now.

Why PassiveIntent

Built for intervention logic, not surveillance plumbing.

Most teams discover friction after the drop-off. PassiveIntent keeps the model close to the session so you can respond while the decision is still in motion.

It is lightweight enough to ship broadly, private enough for sensitive flows, and specific enough to support real product decisions instead of another dashboard.

Open-source core vs enterprise

Start with the pure core. Expand when the journey gets bigger.

The AGPL core is built for one live session on one site in one tab. Start with a Blueprint when the journey is standard and sessions are short. Enterprise is for brands that need continuity across repeat visits, connected properties, customer-specific funnels, and high-volatility campaigns.

FAQ

What teams usually need answered before they adopt.

What is the main job PassiveIntent solves?

Recovering users in the last moments before abandonment while intent is still active.

Do we need to send behavioral data to your servers?

No. Intent evaluation runs locally in the browser by default.

Do we need a cookie banner for PassiveIntent to work?

For the default zero-egress, in-memory deployment, no.* If you enable persistent browser storage or export graph data to your own infrastructure, your consent and privacy obligations depend on your implementation and legal review.

Is this only for engineers?

No. Product, growth, and CX teams can use the signals to drive practical intervention playbooks.

What is the difference between PassiveIntent Core and the Enterprise pack?

PassiveIntent Core is the AGPL open-source engine for one live session on one site in one tab. If a standard journey only needs a faster start, use a Blueprint. Enterprise expands that same privacy-first model for repeat visits, connected properties, customer-specific short-session funnels, and fast-changing traffic conditions.

Can I use standard tiers to wrap PassiveIntent into my own SDK or app?

No. Standard commercial tiers are for internal business use. Agency, OEM, or wrapped-product scenarios require a separate license.
